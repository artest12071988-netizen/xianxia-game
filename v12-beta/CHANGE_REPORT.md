# V14.6 ADMIN UI FIX4 修正報告

## 問題
部分動態後台模組在專業 UI 建立完成後，仍直接追加到 `#adminMain`。CSS Grid 因此把這些卡片當成額外欄位，造成原功能卡片縮在左側、分類選單被推到中央。

## 修正
- 監聽整個 `#adminMain`，不再只監聽內容容器。
- 將後續動態加入的卡片自動吸收至內容區並重新分類。
- 鎖定側欄與內容區的 Grid 欄位。
- 對尚未歸位的最外層節點加入顯示保護。
- 保留 FIX3 的「儲存修改 → 發布正式版」流程。

## 未修改
- Supabase RPC、資料表、欄位 ID、按鈕事件、遊戲數值及發布邏輯均未修改。
